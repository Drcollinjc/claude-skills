# Accumulated Lessons v1.0.0

## Purpose
Central repository of learned patterns and anti-patterns

## Lessons by Category

### Testing
- Pattern: Mock at boundaries, not internals
- Pattern: Use fixtures for complex setup
- Anti-pattern: Over-mocking leads to brittle tests

### Performance
- Pattern: Profile before optimizing
- Pattern: Cache expensive computations
- Anti-pattern: Premature optimization

### AWS
- Pattern: Always tag resources
- Pattern: Use least-privilege IAM
- Anti-pattern: Hardcoded credentials

### Code Quality
- Pattern: Fail fast with validation
- Pattern: Explicit is better than implicit
- Anti-pattern: Silent failures
